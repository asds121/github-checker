# GitHub 访问检测工具 - 增强版

## 概述
一个功能强大的图形化工具，用于检测 GitHub 服务的可访问性，并提供详细的响应时间测量和统计分析。

## 功能特性

### 🚀 核心功能
- **实时访问状态检测** - 支持多URL检测（github.com 和 api.github.com）
- **响应时间测量** - 精确到毫秒的响应时间测量
- **自动重试机制** - 可配置的失败重试次数
- **可视化状态指示器** - 颜色编码的状态显示
- **手动和自动检测模式** - 支持5秒间隔的自动检测

### 📊 增强功能
- **详细日志记录** - 完整的操作日志，支持不同级别
- **统计信息** - 总检测次数、成功率、失败率统计
- **配置管理** - 可配置的设置（超时时间、重试次数、检测间隔等）
- **日志导出** - 支持导出日志文件
- **多URL支持** - 同时检测多个GitHub服务
- **错误处理增强** - 完善的异常处理机制

### 🛠️ 用户界面
- **现代化界面** - 改进的UI设计，支持窗口大小调整
- **菜单系统** - 完整的文件、工具、帮助菜单
- **实时日志显示** - 内置日志查看器
- **进度指示器** - 检测过程中的进度显示
- **设置对话框** - 图形化配置界面

## 技术特性

### 性能优化
- **多线程检测** - 后台线程执行检测，不阻塞UI

### 可靠性增强

## 依赖要求

### 必需依赖
- **Python 3.7+** - 推荐使用 Python 3.8 或更高版本
- **requests** - HTTP请求库
- **tkinter** - GUI库（通常随Python自带）

### 安装依赖
```bash
pip install requests
```

## 使用方法

### 方法一：使用批处理文件（Windows 系统）
双击 `start.bat` 或运行以下命令：
```bash
start.bat
```

### 方法二：直接执行 Python 脚本
```bash
python github_checker.py
```

### 方法三：使用增强启动脚本
```bash
python enhanced_start.py
```

## 配置说明

### 配置文件 (config.json)
```json
{
    "timeout": 10,
    "auto_check_interval": 5,
    "max_retries": 3,
    "log_level": "INFO",
    "window_geometry": "500x400",
    "check_urls": ["https://github.com", "https://api.github.com"],
    "user_agents": [
        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36...",
        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36..."
    ]
}
```

### 配置选项说明
- **timeout**: 请求超时时间（秒）
- **auto_check_interval**: 自动检测间隔（秒）
- **max_retries**: 最大重试次数
- **log_level**: 日志级别（DEBUG, INFO, WARNING, ERROR）
- **window_geometry**: 窗口初始大小
- **check_urls**: 要检测的URL列表
- **user_agents**: 用户代理字符串列表

## 界面说明

### 状态指示器
- **灰色**: 就绪 / 未检测
- **橙色**: 检测中
- **绿色**: 可访问（响应时间显示）
- **红色**: 访问失败 / 出错

### 统计信息
- **总检测次数**: 累计检测总次数
- **成功次数**: 成功访问的次数
- **失败次数**: 访问失败的次数

### 菜单功能
- **文件**
  - 导出日志: 将应用日志导出到文件
  - 退出: 安全退出应用
- **工具**
  - 设置: 打开配置对话框
  - 清除统计: 重置所有统计数据
- **帮助**
  - 关于: 显示应用信息
  - 查看完整日志: 打开日志查看器

## 工作原理

### 检测流程
1. **URL选择**: 从配置中选择要检测的URL
2. **请求发送**: 使用浏览器用户代理发送HTTP请求
3. **响应处理**: 检查响应状态码和内容
4. **时间测量**: 计算精确的响应时间
5. **结果显示**: 在UI中显示检测结果

### 错误处理
- **超时处理**: 请求超时自动重试
- **连接错误**: 网络连接问题的处理
- **HTTP错误**: 各种HTTP状态码的处理
- **异常捕获**: 通用异常的安全处理

### 日志系统
- **多级别日志**: DEBUG, INFO, WARNING, ERROR
- **文件日志**: 详细的操作日志保存到文件
- **UI日志**: 实时的界面日志显示
- **日志轮转**: 防止日志文件过大

## 注意事项

### 网络要求
- 需要稳定的网络连接
- 可能需要配置代理（取决于网络环境）
- 某些网络环境可能需要特殊设置

### 性能考虑
- 建议设置合理的检测间隔
- 避免过频繁的网络请求
- 注意系统资源使用情况

### 安全说明
- 本工具仅用于网络连接性检测
- 不会收集或发送用户数据
- 所有检测均在本地执行
- 日志文件保存在本地

## 故障排除

### 常见问题
1. **无法启动**: 检查Python环境和依赖安装
2. **检测失败**: 检查网络连接和代理设置
3. **界面卡顿**: 调整检测间隔或关闭自动模式
4. **日志错误**: 检查文件权限和磁盘空间

### 调试模式
- 将日志级别设置为DEBUG获取详细信息
- 查看完整的应用日志进行问题诊断
- 使用设置对话框调整配置参数
