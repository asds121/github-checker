# GitHub 访问检测工具
## 概述
一个简单的图形化工具，用于检测 github.com 是否可访问并测量响应时间。

## 功能特性
- 实时访问状态检测
- 响应时间测量（以毫秒为单位）
- 手动检测按钮
- 自动检测模式（5秒间隔）
- 可视化状态指示器（颜色编码）

## 依赖要求
- Python 3.x 版本
- requests 库
- tkinter（通常随 Python 自带）

## 安装步骤
1. 安装所需库：
   ```
   pip install requests
   ```

## 使用方法
### 方法一：使用批处理文件（Windows 系统）
双击 `start.bat` 或运行以下命令：
```bash
start.bat
```

### 方法二：直接执行 Python 脚本
```bash
python github_checker.py
```

## 工作原理
1. 向 https://github.com 发送 HTTP 请求
2. 使用类浏览器请求头（User-Agent、Referer）
3. 设置 10 秒超时时间，避免长时间等待
4. 测量响应时间
5. 在图形界面中显示检测状态

## 状态指示器说明
- **灰色**：就绪 / 未检测
- **橙色**：检测中
- **绿色**：可访问（响应时间 < 10 秒）
- **红色**：访问失败 / 超时 / 出错

## 注意事项
- 本工具仅检测 HTTP/HTTPS 连接性
- 不会打开真实浏览器
- 所有检测均在本地执行
- 不会向外部服务器发送任何数据
