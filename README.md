# GitHub网络状态检测工具

一个智能的GitHub网络连接状态检测工具，帮助用户准确判断当前网络环境下GitHub服务的可用性，为用户提供操作建议。

## 功能特性

### 核心功能
- **智能网络检测**：自动检测GitHub网站和API的网络连接状态
- **多维度分析**：基于延迟、成功率、稳定性等指标进行综合评估
- **智能建议生成**：根据网络状态提供操作建议和预测
- **历史数据管理**：保存检测历史，支持数据清理和统计

### 用户体验
- **简洁界面**：直观的用户界面，状态指示灯显示
- **自动检测**：支持定时自动检测，可自定义检测间隔
- **实时反馈**：检测结果实时显示，包含详细数据信息
- **个性化配置**：丰富的配置选项，支持自定义阈值和参数

### 技术特性
- **插件化架构**：基于微内核的插件化设计，易于扩展
- **跨平台支持**：支持Windows、macOS、Linux操作系统
- **轻量高效**：资源占用少，运行效率高
- **稳定可靠**：完善的异常处理和错误恢复机制

## 系统要求

### 软件要求
- Python 3.8 或更高版本
- 支持的操作系统：
  - Windows 10 及以上版本
  - macOS 10.14 及以上版本
  - Ubuntu 18.04 及以上版本

### 硬件要求
- 内存：2GB 及以上
- 存储空间：100MB 可用空间
- 网络连接：需要能够访问GitHub服务

## 使用方法

### 基本使用
1. 启动应用程序
2. 点击"开始检测"按钮进行网络检测
3. 查看检测结果和建议

### 自动检测
1. 点击"自动检测: 关"按钮开启自动检测
2. 系统将按照设定的间隔自动进行检测
3. 再次点击可关闭自动检测

### 设置配置
1. 点击"设置"按钮打开设置窗口
2. 可以配置以下参数：
   - 检测超时时间
   - 自动检测间隔
   - 数据保留时间
   - 延迟和稳定性阈值
   - 界面主题和语言
   - 网络代理设置
   - 日志级别

### 快捷键
- `Ctrl+D` 或 `F5`：开始检测
- `Ctrl+A`：切换自动检测
- `Ctrl+S`：打开设置
- `F1`：显示帮助
- `Ctrl+Q`：退出程序

## 配置说明

### 检测设置
- **超时时间**：单次检测的最大等待时间（秒）
- **自动检测间隔**：自动检测的时间间隔（分钟）
- **数据保留时间**：历史数据的保留天数
- **检测URL列表**：需要检测的GitHub服务URL

### 阈值设置
- **良好延迟阈值**：延迟低于此值视为良好状态（毫秒）
- **警告延迟阈值**：延迟高于此值视为警告状态（毫秒）
- **稳定性阈值**：稳定性评分的阈值（0-1）

### 界面设置
- **界面主题**：可选择默认、深色、浅色主题
- **界面语言**：支持中文和英文
- **系统托盘图标**：是否显示系统托盘图标
- **启动时最小化**：启动时是否最小化窗口

### 高级设置
- **HTTP代理**：设置HTTP代理服务器
- **验证SSL证书**：是否验证SSL证书
- **日志级别**：设置日志输出级别

## 架构设计

### 总体架构
采用插件化微内核架构，主要包含以下层次：
- **用户界面层**：提供图形用户界面
- **应用服务层**：协调各个服务组件
- **核心业务层**：实现核心功能插件
- **基础设施层**：提供基础服务支持

### 核心组件
- **微内核**：提供插件管理和服务注册功能
- **插件管理器**：管理插件的生命周期
- **事件总线**：实现组件间的松耦合通信
- **配置服务**：统一管理应用程序配置
- **日志服务**：提供日志记录和管理功能
- **存储服务**：提供数据持久化功能

### 插件系统
- **网络检测器插件**：负责网络连接检测
- **数据收集器插件**：负责检测数据收集和管理
- **决策引擎插件**：负责网络状态分析和建议生成

## 故障排除

### 常见问题

#### 1. 检测失败
- 检查网络连接是否正常
- 确认GitHub服务是否可访问
- 检查防火墙和代理设置
- 查看日志文件获取详细信息

#### 2. 程序无法启动
- 确认Python版本是否符合要求
- 检查依赖项是否完整安装
- 查看错误日志获取具体信息

#### 3. 自动检测不工作
- 检查自动检测间隔设置
- 确认程序没有被系统休眠
- 查看日志中的错误信息

#### 4. 界面显示异常
- 尝试切换界面主题
- 检查系统字体设置
- 重启应用程序

### 日志查看
日志文件默认保存在 `logs/detector.log`，可以通过设置调整日志级别来获取更详细的信息。
